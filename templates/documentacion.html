1 Arquitectura de Software y Topología

La plataforma ha sido diseñada utilizando el patrón arquitectónico Cliente-Servidor estructurado en tres capas lógicas. Este diseño asegura una alta cohesión, bajo acoplamiento y escalabilidad horizontal en entornos de producción.

Capa de Presentación

Construida con HTML5 semántico, utilidades CSS mediante Tailwind CSS y control del DOM con JavaScript. Las vistas son renderizadas de forma dinámica en el servidor (SSR) mediante el motor de plantillas Jinja2.

Capa de Lógica (Backend)

Desarrollada en Python utilizando el microframework Flask. El enrutamiento, validación de sesiones y reglas de negocio se ejecutan aquí. El despliegue se gestiona en Render.com empleando Gunicorn (WSGI).

Capa de Persistencia

Almacenamiento relacional operado por un clúster de MySQL alojado en Aiven.io. La comunicación desde el backend se realiza mediante mysql-connector-python aplicando cifrado TLS y variables de entorno.

2 Modelo Entidad-Relación (MER)

El esquema relacional está normalizado hasta la Tercera Forma Normal (3FN) para garantizar la integridad referencial y evitar anomalías de inserción, actualización o borrado. El núcleo operativo vincula cinco entidades primarias.

  • La tabla profesores y materias representan catálogos independientes.
  • La tabla asignaciones actúa como una entidad pivote que resuelve la cardinalidad de muchos a muchos (N:M) entre profesores y materias, especificando además la sección impartida.
  • La tabla estudiantes mantiene los expedientes físicos de los alumnos matriculados.
  • La tabla calificaciones vincula directamente al estudiante con una asignación específica, asegurando mediante un constraint UNIQUE que un estudiante no posea dos notas finales para la misma asignación.

4 Reglas de Negocio y Seguridad (Backend)

Límites Académicos Estrictos

El backend implementa interceptores de consultas para proteger la carga académica:

  • Tope Docente: Se ejecuta una instrucción COUNT() previa a la inserción. Si el resultado es ≥ 2 asignaciones activas para un mismo profesor, se bloquea la transacción.
  • Tope Estudiantil: Ningún estudiante puede tener vinculados más de 6 registros en la tabla de calificaciones, previniendo la sobrecarga de matrícula.

Control de Acceso y Auditoría

El sistema valida el rol almacenado en la sesión de Flask en cada petición protegida:

  • Modo Auditoría: El rol 'admin' posee delegación global para ajustar o revocar calificaciones de cualquier curso.
  • Aislamiento Docente: El rol 'profesor' sufre un filtrado automático a nivel de consulta SQL, visualizando exclusivamente las tablas relacionales de sus materias asignadas.

Criptografía y Sanitización

La integridad de los datos de acceso está garantizada mediante la librería werkzeug.security. Toda contraseña plana que ingresa al controlador POST es convertida de inmediato en un hash mediante el método scrypt, imposibilitando la ingeniería inversa incluso si la base de datos es compromised. Previo al hash, el servidor valida estrictamente que la longitud de la cadena plana sea mayor o igual a 5 caracteres. Adicionalmente, los campos de nombre son filtrados mediante Expresiones Regulares (RegEx) para denegar el registro de caracteres numéricos.

5 Guía de Despliegue (DevOps)

El proyecto está diseñado para integración y despliegue continuo (CI/CD) vinculando el repositorio directamente a los Web Services de Render.

Dependencias del Entorno

El empaquetado del entorno virtual se define en el archivo requirements.txt, requiriendo de forma obligatoria las librerías: Flask, mysql-connector-python, Werkzeug y gunicorn==22.0.0 para el levantamiento en producción.

Comandos de Inicialización

Build Command: pip install -r requirements.txt
Start Command: gunicorn app:app

Gestión de Secretos

Por normativas de seguridad, las credenciales de Aiven no se exponen en el código fuente. El archivo principal extrae la clave de la base de datos dinámicamente utilizando la instrucción os.environ.get('DB_PASSWORD'). Esta variable debe ser inyectada directamente en la pestaña Environment del panel de control de Render.
